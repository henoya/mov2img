# mov2img

画面録画から静止画像を自動抽出するBashスクリプトユーティリティです。このツールは動画ファイル（MOV/MP4）を解析し、画面が静止している瞬間を検出して、それらのフレームをPNG画像として抽出します。操作マニュアルやドキュメント作成に最適です。

[English README](README.md)

## 機能

- **自動静止フレーム検出**: 画面が変化していない瞬間をインテリジェントに識別
- **調整可能な閾値**: 画面変化を検出する感度を調整可能
- **バッチ処理**: 1つの動画から複数の静止フレームを抽出
- **カスタマイズ可能な出力**: 出力ディレクトリとファイル名パターンを設定可能
- **フレームレート制御**: 最適なパフォーマンスのために好みのフレームレートで処理
- **詳細モード**: デバッグと進行状況監視のための詳細ログ出力

## 必要要件

- macOS / Linux（Bash環境）
- `ffmpeg` と `ffprobe`（動画処理用）
- `bc`（浮動小数点計算用）

### macOSでのインストール

```bash
brew install ffmpeg
```

## 使い方

### 基本的な使い方

```bash
./mov2img.sh -i recording.mov
```

これにより`recording.mov`から静止フレームを抽出し、`recording_frames/`ディレクトリに保存します。

### 高度な使い方

```bash
# 出力ディレクトリとファイルプレフィックスを指定
./mov2img.sh -i tutorial.mp4 -o output_frames -n screenshot

# 検出感度を調整（値が小さいほど感度が高い）
./mov2img.sh -i demo.mov -t 2

# 異なるフレームレートと最小継続時間で処理
./mov2img.sh -i presentation.mov -f 15 -d 0.5

# デバッグ用の詳細出力を有効化
./mov2img.sh -i recording.mov -v
```

### パラメータ

| パラメータ | ロングオプション | 説明 | デフォルト値 |
|-----------|-----------------|------|-------------|
| `-i` | `--input` | 入力動画ファイル（必須） | - |
| `-o` | `--output` | 出力ディレクトリ | `{入力ファイル名}_frames` |
| `-n` | `--name` | 抽出画像のベース名 | `{出力ディレクトリ名}` |
| `-t` | `--threshold` | 差分閾値パーセント | `3%` |
| `-f` | `--fps` | 処理フレームレート | `30` |
| `-d` | `--duration` | 最小静止時間（秒） | `1.0` |
| `-v` | `--verbose` | 詳細出力を有効化 | `false` |
| `-h` | `--help` | ヘルプメッセージを表示 | - |

## 動作の仕組み

1. **フレーム抽出**: 指定されたフレームレートで入力動画からフレームを抽出
2. **フレーム比較**: 連続するフレームを比較して差分を検出
3. **静止期間の検出**: フレーム差分が最小継続時間の間、閾値を下回ると静止期間として識別
4. **画像エクスポート**: 各静止期間の中央フレームをPNG画像として保存

## ワークフロー例

1. アプリケーションやチュートリアルの画面録画を作成
2. mov2imgを実行して重要な瞬間を抽出：
   ```bash
   ./mov2img.sh -i screen_recording.mov -n step
   ```
3. `screen_recording_frames/`に抽出された画像を確認：
   - `step_0001.png`
   - `step_0002.png`
   - `step_0003.png`
   - ...


## 使用のヒント

- **チュートリアル用**: 微細なUI変化をキャプチャするため低い閾値（1-2%）を使用
- **プレゼンテーション用**: 小さなアニメーションを無視するため高い閾値（5-10%）を使用
- **モバイル録画用**: パフォーマンス向上のためFPSを15-20に調整
- **長時間の動画用**: 進行状況を監視するため詳細モードを使用

## トラブルシューティング

### フレームが抽出されない場合
- `-t 1`で閾値を下げてみる
- `-d 0.5`で最小継続時間を短縮
- 動画に最小継続時間以上の静止期間があるか確認

### フレームが多すぎる場合
- `-t 5`で閾値を上げる
- `-d 2`で最小継続時間を延長

### パフォーマンスの問題
- `-f 15`で処理フレームレートを削減
- 一時フレーム保存用の十分なディスク容量を確保
